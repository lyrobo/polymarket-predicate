# Polymarket 双边限价单策略 — 完整设计方案

> 目标：在 Polymarket 加密涨跌预测市场，通过同时挂双边极限低价限价单，实现统计套利。

---

## 一、市场机制理解

### 1.1 代币模型

```
每个预测市场（如 BTC 15m）有两个代币：
┌─────────────────────────────────────┐
│  UP Token    → BTC 涨 → 到期 $1.00  │
│  DOWN Token  → BTC 跌 → 到期 $1.00  │
│                                      │
│  必然：其中一个归 $1.00，另一个归 $0  │
│  所以：UP + DOWN = $1.00 (在任何时刻) │
└─────────────────────────────────────┘
```

### 1.2 套利原理

```
如果 UP 卖 $0.05 且 DOWN 卖 $0.05：
  同时买入各 100 份
  成本: 100×$0.05 + 100×$0.05 = $10.00
  到期回报: 100×$1.00 = $100.00（赢的那边）
  净利润: +$90.00
  
ROI = 900%
```

### 1.3 为什么这个机会存在

| 原因 | 说明 |
|------|------|
| **市场碎片化** | UP 和 DOWN 在不同订单簿，没人同时看两边 |
| **恐慌抛售** | 价格急剧变化时，一边被砸到地板价 |
| **流动性真空** | 15m 窗口末期，做市商撤单，散户乱挂 |
| **开盘价失真** | 新窗口刚开盘，两边价格可能同时很低 |

## 二、策略参数设计

### 2.1 三层限价体系

```
┌──────────────────────────────────────────────────┐
│ 层级    限价      份数    成本     回报    利润   │
├──────────────────────────────────────────────────┤
│ 🥇 激进  $0.01   100sh   $2.00   $100   +$98    │
│ 🥈 稳健  $0.05   100sh   $10.00  $100   +$90    │
│ 🥉 保守  $0.10   100sh   $20.00  $100   +$80    │
│                                                  │
│ 触发条件：两边价格都 ≤ 对应限价                    │
│ 且 UP_price + DOWN_price < 0.98（保证>2%利润）    │
└──────────────────────────────────────────────────┘
```

**推荐先用 🥉 保守层**，成交概率最高；逐步下调。

### 2.2 市场选择

| 资产 | 时间框架 | 理由 |
|------|---------|------|
| BTC | 15m, 1h | 流动性最好，深度最大 |
| ETH | 15m, 1h | 与 BTC 高度相关但独立机会 |
| SOL | 15m | 波动大，极端价格更常见 |
| XRP | 15m | 同上，但流动性差 |

> 初期只做 BTC+ETH 15m（4个市场），稳定后再扩展。

### 2.3 时机选择

```
窗口生命周期：
├─ 0-3 min   ← 开盘震荡，价格波动大，机会多但方向不明
├─ 3-10 min  ← 价格稳定，做市商活跃
├─ 10-14 min ← 窗口末期，流动性枯竭，极端价格出现 ⭐ 最佳
└─ 14-15 min ← 几乎无流动性
```

**重点监控窗口最后 5 分钟**，此时最可能出现 $0.01-$0.10 的极端价格。

## 三、风控体系

### 3.1 最大敞口

```
最大同时配对组数:  3 组
每组最大投入:      $20 (保守层) / $10 (稳健) / $2 (激进)
总最大敞口:        $60

占本金比例:
  本金 $50  → 120%（太激进）
  本金 $100 → 60%（可接受）
  本金 $200 → 30%（安全）
  
→ 推荐起始本金 ≥ $100
```

### 3.2 部分成交处理

```
场景：UP 成交 100 份 @ $0.05，但 DOWN 只成交 30 份

处理方案：
├─ 等待 X 秒看 DOWN 是否继续成交
├─ 如果超时未满 → 取消未成交的 DOWN 单
├─ 同时市价卖出多余的 UP（亏损卖出差额）
├─ 记录为"残废配对"→ 亏损 = 多买的 UP 成本 - 卖出价

紧急情况：
├─ 如果 UP 成交了但 DOWN 完全没成交
├─ 且窗口只剩 <2 分钟
└─ → 立即市价卖出所有 UP，承担全部亏损
```

### 3.3 资金锁定风险

```
双边挂单后，资金被锁定：
├─ UP 限价单:    $1.00-$10.00 被锁定
├─ DOWN 限价单:  $1.00-$10.00 被锁定
└─ 总锁定:       $2.00-$20.00

如果长期不成交（几小时）→ 资金利用率低
→ 设置订单超时: 300s 不成交自动取消
→ 避免资金被锁死在不可能成交的价格上
```

## 四、出场策略

### 4.1 完美配对 → 持有到结算

```
✅ 两边都成交 → 获得 UP + DOWN 双份
   → 持有到窗口结算
   → 赢家自动赎回 $1.00/份
   → 无需任何操作，稳赚
```

### 4.2 价格反转 → 提前平仓

```
如果配对后，市场价格大幅变化：
├─ UP 从 $0.05 涨到 $0.50
├─ DOWN 从 $0.05 涨到 $0.50
└─ 此时 UP+DOWN = $1.00 → 可以直接卖出一对，提前锁定利润

触发条件：配对后任何时刻，如果：
  当前 UP + 当前 DOWN ≥ 买入价 × 1.5
→ 立即卖出双边，提前落袋
```

### 4.3 单边意外 → 紧急止损

```
如果只有一边成交，另一边迟迟不成交：
├─ 超时: 120s
├─ 措施: 
│   ├─ 取消未成交单
│   ├─ 市价卖出已成交的一边
│   └─ 记录亏损
└─ 亏损上限: 单边成本 = $1-$10
```

## 六、关键风险与对策

### 6.1 流动性幻觉

```
风险: 看到 $0.01 的价格，但只有 1-2 份深度
      → 100 份限价单永远不可能成交

对策:
├─ 挂单前检查订单簿深度
├─ 只挂「最佳买价 ± 1 tick」的限价单
└─ 如果 30s 未成交 → 取消，不浪费锁定资金
```

### 6.2 市场提前关闭

```
风险: Polymarket 偶尔提前关闭有争议的市场

对策:
├─ 监控 market.active 状态
└─ 如果市场关闭时持有单边 → 按 $0 或 $1 结算
```

### 6.3 API 限流

```
风险: 频繁查询订单状态触发 rate limit

对策:
├─ 缓存价格 (1s TTL)
├─ 订单状态查询降频 (每 3 秒而非每轮)
└─ 指数退避重试
```

### 6.4 滑点

```
风险: 限价单 $0.05 成交了，但实际支付了更高价格
      (Polymarket 的撮合可能按对手价成交)

对策:
├─ 使用 Limit Order (GTC)，不用 Market Order
├─ 成交后校对实际价格
└─ 如果实际价格偏离 > 10% → 标记异常
```
